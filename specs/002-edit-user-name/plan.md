# Implementation Plan: Edit user name in popup

**Branch**: `002-edit-user-name` | **Date**: 2026-04-30 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-edit-user-name/spec.md`

## Summary

Enable inline editing of a user's name inside the user details popup by clicking the current name. The flow includes save/cancel controls, whitespace validation, optimistic UX feedback for loading, error handling for failed updates, and explicit conflict handling for concurrent edits requiring refresh/reopen before retry.

## Technical Context

**Language/Version**: TypeScript 6.x (strict), React 19, Vite 8  
**Primary Dependencies**: `react`, `react-dom`, `@supabase/supabase-js`, `@tanstack/react-query`  
**Storage**: Supabase Postgres (`public.users`)  
**Testing**: Vitest + Testing Library (`npm run test`)  
**Target Platform**: Modern evergreen browsers (desktop-first)  
**Project Type**: SPA (Vite + React)  
**Performance Goals**: Enter edit mode in under 100 ms and complete successful save round-trip within typical network latency; no full page reload needed  
**Constraints**: API access only through `src/api/`; server state and mutations via React Query hooks; lint/format compliance required  
**Scale/Scope**: Single-user name edits from popup only; no bulk editing, no schema changes
